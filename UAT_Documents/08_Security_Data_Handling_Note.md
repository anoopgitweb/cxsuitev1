# CX Intelligence Suite - Security and Data Handling Note

This note is for UAT testers and account data owners.

## 1. Local Processing

CX Intelligence Suite Version 24.6 is designed as a local toolkit.

Observed package behavior:

- The launcher starts a local server.
- The browser opens the app from `http://127.0.0.1:8765/`.
- The package includes `portable_python`, so installed Python is not required.
- App files, models, logs, and outputs stay inside the suite folder structure.

## 2. Data Location

During UAT, account files should be handled according to account data rules.

Local generated artifacts may include:

- Uploaded file processing traces
- Analysis outputs
- Exported Excel workbooks
- Board Room HTML/PDF output if generated by the tester
- Server logs
- Audit logs
- User logs

## 3. Logs

The audit readme states the app records:

- Server start
- Login success/failure
- File uploads
- Analysis start/completion/failure
- Exports
- Manual override actions
- Important UI clicks from suite pages

Log locations:

- `logs\server.log`
- `logs\audit`
- `logs\user_logs`

## 4. User Management

The audit readme states first launch creates `admin / admin123`.

UAT control:

- Change default admin password before broad UAT.
- Share access only with approved testers.
- Record tester names in the participation matrix.

## 5. Data Minimization

Before UAT:

- Remove fields not needed for analysis.
- Avoid unnecessary personal or sensitive data.
- Use account-approved test files.
- Do not email raw customer files unless approved by the account data owner.

## 6. Output Handling

Exported files should be treated as account data because they may include:

- Customer verbatims
- Agent and manager names
- Scores
- Sentiment labels
- Themes
- Evidence tables

Store and share exports only through approved UAT channels.

## 7. Issue Evidence

When logging a defect:

- Include screenshots only when permitted.
- Mask customer-identifying text if required.
- Prefer row counts, column names, and error messages over raw customer comments.
- Reference `logs\server.log` only when needed for troubleshooting.
